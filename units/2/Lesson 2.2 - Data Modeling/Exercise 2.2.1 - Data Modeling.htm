Exercise 2.2.1

Data Modeling

Data Modeling is when you look at a problem and figure out what data structures are needed.

We are going to practice this by looking at web sites to see what information they store.

How do you know what information a web site stores? Think about how the data of web site changes as users use it. What changes? The things that change due to user actions are the things that are (probably) stored by the web site.

Instructions

  • Work in groups of 2-3.

  • Choose a web site that you are familiar (and have an account with) with so you do not have to spend time learning what the site does or signing up.

  • Look for data that is stored by the web site. A good hint that the site is storing some data is if the site changes as users use the site. It is OK to make your best guess if you are not sure whether some information is being stored or not.

  • Write in English what is being stored. You can do this using comments at the top of a Javascript file. For example:

    somesite.js
    // A list of users

    // For each user:
    //     First Name
    //     Last Name
    //     Email
    //     Password

    You don't have to do this for all the data on the site if the site is complicated, but see how many different types of data you can find.

  • Next, write a Javascript example that shows all the different data structures that you found. For example:

    • somesite.js
    // A list of users

    // For each user:
    //     First Name
    //     Last Name
    //     Email
    //     Password

    var users = [
        {
            firstName: "Bob",
            lastName: "Smith"
            email: "bobs@example.com"
            password: "abc123"
        }
    ];

  • You don't have to use real data from the site, just try to make your example be similar to what would appear on the site. If you are having trouble with some Javascript syntax, use The Guide. Make sure your Javascript file loads in Chrome without any errors.

  • Submit your Javascript file at the end of the period. Everyone in a group should submit a file.
