Input

The human touch

In this lesson, we will learn how to have our programs react to a person using a web page. We will be working with the following types of input:

  • Mouse/trackpad clicking and movement
  • Keyboard input
  • Resizing of the browser window
  • Touch input (if you have a touch device)

Events

When person uses the mouse, trackpad, touches, or types a key, events are recorded for different elements of the page.

When these events happen, we can have Chrome call a function we write. This is called handling an event.

This example handles different events and displays information about them on the canvas (large rectangle) when the events happen.

Events happen for a particular element of the page.

Mouse/trackpad events

The following events happen when a person uses a mouse:

Event Meaning
onmousedown When a mouse button is pressed down while the mouse is in the element
onmousemove When a mouse pointer is moved while in the element
onmouseup When a mouse button is released while the mouse is in the element
onclick When a mouse button is pressed down and released while the mouse is in the element

Try using the mouse in the canvas area of the example and watch the events.

Key events

Keyboard input is sent to the element that has focus, which is "where am I typing". A focused element is usually drawn differently, for example:

The following events happen when someone uses the keyboard:

Event Meaning
onkeydown When a key is pressed or repeats
onkeyup When a key is released
onkeypress When a key is pressed (or repeats) and the character is a regular printable character
onchange After the text in an a text element has been changed and it no longer has the focus.

Try typing in the name and weight text areas watch the events.

Resize events

When the user resizes the browser window, there is a resize event. This event happens for the window object.

Event Meaning
onresize When the browser is being resized by the user.

Try resizing the window and watch the events.

How to handle events

In order to handle an event, you need to know:

  • Which element?
  • Which event?

The steps to handle an event are very similar for all events, and are shown below. Note that the placeholders (like someVariable) need to be filled in by you.

function start() {
    var someVariable = document.getElementById("someId");
    someVariable.someEvent = someFunction;
}

You need to write a function for each event you want to handle. This function has a single parameter, an event object, which contains additional information about what happened.

function someFunction(event) {
    do something here!
}
