The Weather Oracle - Part 2

Ok. You can now make a program that will decide if you're going to do your homework or go outside based on the weather today.
To do this you used condition statements, boolean values and if statements.

if (condition) {
    statements;
}

Multiple Conditions

However, what if you like to do more than one activity when it's sunny?
If I'm in Seattle and it's sunny, than I'll go for a bike ride.
However, if I'm in Hawaii and it's sunny, then I'll go surfing!
Otherwise, if it's sunny, I'll go and read in a park.

Boolean Operators

There are three Boolean Operators that are useful to evaluate expressions with with more than one condition.
Two are the and and the or operators.
In Javascript they are written with && and || respectively.
They work exactly like you'd expect. If it's sunny and it's a weekend then I am going biking.

var isSunny=true;
var isWeekend=true;
if (isSunny && isWeekend) {
    activity="go biking";
} else if (isSunny) {
    activity="go to park";
} else {
    activity="do homework";
}

Here is what the truth table looks like for these operators.

Operator Expression A B Result
and A && B true true true
and A && B true false false
and A && B false true false
and A && B false false false
Operator Expression A B Result
or A || B true true true
or A || B true false true
or A || B false true true
or A || B false false false

When using boolean operators and logical expressions, creating a truth table can help you to understand how these expressions get evaluated.

Then there is also the not operator. The not operator uses an exclamation mark (!).
Placing this directly in front of a boolean value negates, or switches its value.

This operator also works exactly like you'd expect.
I like being outside when it's sunny. But, if it's not sunny then I'm going to stay home and play video games.
Now if only that were an option...

if (!isSunny) {
    activity="play video games";
} else {
    activity="do something outside";
}

Here is what the truth table looks like for the not operator.

Operator Expression A Result
not !A true false
not !A false true
