UPrep planning exercise

In this exercise, we are going to create a solution to the following (hypothetical) problem:

The UPrep staff are planning the 2014/2015 school year and have some estimates for how many students would be in each grade. They may want to know the average number of students per grade for the whole school.

Averages

You probably know from math that the average is the sum of the numbers divided by the count of the numbers. For example, to average the numbers:

2, 3, 7

We first total the numbers:

2 + 3 + 7 = 12

Then we divide the total by 3 (since there are three numbers):

12 / 3 = 4

Lets make a program!

To solve this problem we will create a program. A program is just a set of Data Structures and Algorithms.

There are many ways to construct a program. Today we are going to use a programming language called Javascript to construct our program. In our program, we will create an algorithm to solve our problem.

Recall that an algorithm is:

  • Starts with pieces of data (inputs)
  • Produces pieces of data (outputs)
  • Has a well-defined set of steps for computing the answer

You may also recall that algorithms can also use additional data structures while they are computing the result. We will call those temporary data structures.

Lets make a plan

When a programmer starts working on a solution to a problem, they often start out by taking notes about what approach they will take to the solution. They create a plan.

We'll alwys make a plan for our problems by answering the following questions:

  • What is the input data?
  • What is the output data?
  • What are the steps in the algorithm?
  • What temporary data does the algorithm need?

What is the input data?

The UPrep staff have given us our input data, which is a count of students for each grade at UPrep (for the 2013/2014 year).

Grade Number of Students
6th grade 66
7th grade 74
8th grade 74
9th grade 82
10th grade 81
11th grade 79
12th grade 76

What is the output data?

The output is a single number: the average students per grade for the school.

What are the steps for the algorithm?

Our algorithm has two steps:

  1. Calculate the total students for the school by adding together the number of students in each grade.
  2. Calculate the average students per grade by dividing the total students by 7 (the number of grades in the school).

When we write down an informal description of an algorithm (like the above), we call the description pseudocode.

What temporary data does the program need?

The second step of our algorithm needs the total students from the first step. That calculation is not part of the input or output, but is necessary for the algorithm, so it is temporary data.

Turning the plan into a program

Now that we have a plan, lets create a program using Javascript.

  • We will use variables to define our data structures.
  • We will use statements to define the steps of our algorithm.
  • We will use functions to group our statements.

You might be wondering, how much of what I'm learning in this class is specific to the Javascript programming language? Am I going to be able to use any of what I'm learning for other programming languages? Many (but not all) programming languages have the concepts of variables, statements, and functions. There is a good chance that what you learn in this class about Javascript will help you learn other languages more easily.
