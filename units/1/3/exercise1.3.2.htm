In this class, we have had a number of exercises where you have been knee-deep in the Javascript and not even a single thing showed up on the screen. Things are about to change...

In this series of exercises, we are now going to use a web page for the output of an algorithm. Remember this?

The output of an algorithm doesn't just have to sit in a variable once we have calculated it. We can send it to the screen, send it to a 3d printer, send it to another computer, ...

The assignment

We are going to make the following page for the UPrep staff. It shows the number of students in each grade.

The plan

The above picture is the final product. We need to take things step by step. The overall plan is as follows:

  1. Create an HTML page that just uses fake student numbers.
  2. Style the page using CSS.
  3. Add Javascript that fills in the actual student numbers.

We will do steps 1 and 2 in this Exercise, and step 3 in the following exercises.

It is common to have a plan like this when creating web sites. First, you might make a website that looks nice, but uses fake data and does not actually work (often called called a design mockup). When you are happy with the design of the site, and you know what the site is supposed to do, then you actually write the Javascript and make the site work correctly. The job of creating the design mockup may done by someone that knows CSS and HTML (and Graphic Design), but may not know Javascript.
