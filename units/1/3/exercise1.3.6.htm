Character Codes

Copyright

Have you ever come across some characters on a web page that are not on your keyboard? For example, the copyright symbol at the bottom of https://www.facebook.com/?

These characters are not on your keyboard, but they are still characters, and we can still use them in our HTML and our Javascript. We just need to know their character code.

Character Codes

Every character that you can use in a computer has a character code (a number). This means that text in a computer is really just a series of numbers in a row, but the computer knows that these series of numbers meant to be used as characters and not numbers.

To see this in action, go to the following web site and type some text.

http://ascii.co.uk/interactive

For example:

You should see a series of boxes appear that have the number (the character code) of each of the characters you typed.

The character code is also shown in base 16 (Hex, short for hexadecimal) and base 8 (Oct, short for octal). Programmers sometimes like to use different number bases.

Who decides?

But who picks the character code for a character? Who decides that an U is an 85, a P is an 80, and so on?

Whatever numbers we pick, we better all agree, otherwise when one computer reads the web page that was made by another computer, it might get confused.

We need a table

So we really need a table that shows us what character codes go with what characters.

One of the first such tables (finalized in 1963) had only 128 different characters. It was called ASCII, which is short for American Standard Code for Information Interchange.

You can see part of the table here: http://ascii.co.uk/table

We need a bigger table

When you looked at the ASCII table, you may have noticed that the letters are mostly english. What about the rest of the languages in the world?

In 1991, the first version of a standard called Unicode was finalized. This version of Unicode allowed for up to 65535 characters. This larger range of character codes (larger table) allowed for characters used in many more written languages. Since 1991, even more characters have been added.

Take a look: http://www.unicode.org/charts/. You will find Scripts(writing systems) at the top, and Symbols and Punctuation at the bottom.

You might want to bookmark this page, if you want to use any of the unicode special characters in this class.

As you can see, there is quite a variety! Here are some examples:

  • Arabic
  • Mongolian
  • Egyptian Hieroglyphs
  • Cherokee
  • Miscellaneous Symbols (snowman!)
  • Braille Patterns
  • Arrows
  • Emoticons
  • and of course, ASCII

And if you look under Punctuation / General Punctuation / Latin-1 Punctuation, you will find the copyright symbol:

The number below the character (00A9) is the character code shown in hexadecimal (base 16, or hex for short).

So how do we use these characters?

If we have the character code in hex, we can write these characters in both Javascript and HTML.

For example, to write the copyright symbol (00A9 in hex) followed by the year in HTML, we would write:

<p>&#x00A9; 2013</p>

© 2013

To write the same thing in Javascript, we would write:

var copyright = "\u00A9 2013";

You might notice that some of the character codes have 5 hex digits, which means we can't use them with Javascript.

But there's a catch...

The UNICODE standard has a lot of different characters.

But remember, there are different fonts, too. For example, the character A looks different with different fonts.

A A A A A A A A A A A

So a font is really a table as well. The table says what picture to use for each character in that font.

And it turns out that not every font has a picture for every character. For example, if you use one of the Braille Patterns they may or may not work depending on what font you use.

If you see a character like this:

It usually means that there is no picture for that character code in that font.

So, how do you know what font works for what characters? If you search for the words font, unicode, and the hex characters of the character, you may find a list of fonts. Of course, you may not have that font on your computer.
