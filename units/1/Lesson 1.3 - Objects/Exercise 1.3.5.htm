Exercise 1.3.5

Your own objects

In the Exercises in this lesson, we used objects for the number of students in a school, and for points on a line. But the sky is the limit! You can use objects for any data that is related.

For this assignment, think of an object in the real world that you would like to be able to store information about in a computer. It can be anything. Think about what properties you want to store (which might not be all the properties of the real world object).

For example, if you wanted to store a cat object, you would have to figure out what parts of a cat you cared about. Fur color? Eye color? Name? Age?

Then, think about what functions you would have for that object. Those function might be like the real world actions that might apply for that object. What can happen to your object that would change the properties?

Finally, try to write down some examples of these objects, and use them with functions you write.

What about text?

So far, we have been working with number values (and object values). Many real world objects have properties that are not numbers. If you want your object to have properties that are text, you can place the text between double quotes to write what Javascript calls a string value:

"text"

For example:

var pawsTheCat = {
    age: 6,
    name: "paws",
    eyeColor: "brown",
    furColor: "gray"
};

We are going to learn more about string values and text in the next lesson. (If you get stuck, you can find more information here).

Object as property value?

A value of a property of an object can be another object. For example, if the cat paws considers the neighbor dog bruno to be an enemy, then we can write something like:

var brunoTheDog = {
    age: 3
};

var pawsTheCat = {
    age: 6
    enemy: brunoTheDog
};

If we drew the above as a diagram, we would have:

If we wanted to see the age of the enemy of paws (3), we could write:

var pawsEnemy = pawsTheCat.enemy;
var pawsEnemyAge = pawsEnemy.age;

Or just:

var pawsEnemyAge = pawsTheCat.enemy.age;
