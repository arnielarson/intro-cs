Looking ahead

The UPrep staff love the calculations you have been doing for them. But they want more. Does this surprise you?

Once you create a program for someone else to use, they may ask you to add new capabilities (often called features) to your program to make it better.

The UPrep staff are starting to look forward to next year, and have some estimates of how many students will be at UPrep. Their estimate is that for next year, the school population will change in the following way compared to this year:

Grade Change in
Students
6th grade -2
7th grade -3
8th grade -1
9th grade 3
10th grade 1
11th grade 2
12th grade 1

They would like you to calculate average students per grade for the whole school, middle school, and upper school for their next year estimate.

Creating a plan

Even though we are just making a change to our program, we still need a plan.

Lets look at what we already know:

  • We already know the algorithm for calculating the averages for any year.
  • We already have the student data for this year.

We also have some new input data:

  • We have the estimated change for each grade.

Given this, we should be able to add a step to our algorithm that adjusts the input data for this year so we can use it for next year. The outline of our algorithm is then:

  • Using the change data, adjust the data for this year in order to the calculate next year data.
  • Calculate the averages with the next year data.

Using our plan

Now that we have a plan, we can use it to update our existing program.

We identified a new step in our algorithm that adjusts the input data for this year to produce last year. We can create a new function (adjustYear) for that step. There is no reason to limit our function to a particular year, so we can make it work for any year by using a year parameter as we have in the past.

Changing the value of a property

But wait a minute.

function adjustYear(year) {
    How do we change the year values?
}

Our adjustYear function needs to change the value of the properties (grade6, grade7, etc) in the object referenced by the year property. How do we do that? So far we have been using the value of properties, but we have not been changing them.

When we change the value of a property, we have to answer the following:

  • Which object?
  • Which property?
  • What is the new value?

To change the value of a property, we write:

object.property = expression;

For example, if we wanted to change (increase) the value of the grade6 property of a year object by 2, we would write:

year.grade6 = year.grade6 + 2;

If we update our adjustYear function to change the value of each property by 2, we have:

function adjustYear(year) {
    year.grade6 = year.grade6 + 2;
    year.grade7 = year.grade7 + 2;
    year.grade8 = year.grade8 + 2;
    year.grade9 = year.grade9 + 2;
    year.grade10 = year.grade10 + 2;
    year.grade11 = year.grade11 + 2;
    year.grade12 = year.grade12 + 2;
}

Defining the change object

But as you recall, we are not using a value of 2 as the change for all grades. We have a specific change for each grade.

Writing the object value and variable for the changes for each grade, we have:

var change = {
    grade6: -2,
    grade7: -3,
    grade8: -1,
    grade9: 3,
    grade10: 1,
    grade11: 2,
    grade12: 1
};

Using the change object

We can add a change a parameter to adjustYear that lets us provide the changes to make when we call the function. This will allow us to use the function with different change objects.

Updating the function to use the values from the change object (instead of the 2), we have:

function adjustYear(year, change) {
    year.grade6 = year.grade6 + change.grade6;
    year.grade7 = year.grade7 + change.grade7;
    year.grade8 = year.grade8 + change.grade8;
    year.grade9 = year.grade9 + change.grade9;
    year.grade10 = year.grade10 + change.grade10;
    year.grade11 = year.grade11 + change.grade11;
    year.grade12 = year.grade12 + change.grade12;
}

Adding the call to adjustYear

Now we need to add the call to adjustYear that changes the values of the year object. The adjustment to year needs to be made before we calculate the averages. Where should we put the call (red or blue)?

function calculateAverageStudents() {
    var year = {
        grade6: 66,
        grade7: 74,
        grade8: 74,
        grade9: 82,
        grade10: 81,
        grade11: 79,
        grade12: 76
    };
    
    var change = {
        grade6: -2,
        grade7: -3,
        grade8: -1,
        grade9: 3,
        grade10: 1,
        grade11: 2,
        grade12: 1
    };    
    
    adjustYear(year, change);
    var nextYearAverages = calculateAverageStudentsForYear(year);
    adjustYear(year, change);
}
Move your mouse here to see the answer

The location in red is the location we want. The call to adjustYear will change the property values in the year object, and we want that to happen before we calculate the averages using the year object.

Step over

We are familiar with how the average calculations are made, so we can tell Chrome to step over the calls in the next few statements. When we step over, it does not change how the program behaves. It just tells Chrome that we are not interested in stepping through the functions that are being called.
