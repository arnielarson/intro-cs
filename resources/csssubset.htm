Introduction

Cascading Style Sheets (CSS) can be used to provide styling information for an HTML page.

A CSS file is a resource that is used by an HTML page.

Where to put CSS Files and how to name them

Name each file using a short name that describes what the file is for. Avoid spaces in the name. Add .css to the end you choose. Put all the CSS files in the same folder as the HTML files. For example:

default.html

space.css

What a CSS file should look like

A CSS file contains a set of styles, one after another. Each style has a name and properties. For example, the following CSS file (space.css) has two styles: one named spaceship and one named moon.

space.css
.spaceship {
    color: red;
    font-size: 24pt;
}

.moon {
    color: blue;
    font-size: 9pt;
}

The properties are written one after another between the { and the } of the style.

In the above example, the spaceship style has two properties:

  • The color property has the value red, meaning the text should be red.
  • The font-size property has the value 24pt, meaning the text should be 24 points in size (a point is about 1/72nd of an inch).

Making your own styles

When choosing names of styles, pick names that describe the purpose of the style. If a name requires multiple words, then put all the words together (no spaces), but only capitalize the first letter of the second and following words. For example:

mainSpaceship

You can have as many styles as you would like. There are many properties to choose from. Check out the Table Of Contents for a guide.

How to use a CSS file with an HTML file

There are two steps to using a CSS file from an HTML file:

  • Reference the CSS file from the HTML file using a Link Element.
  • Use the Class Attribute for each element that you want to style. The value of the Class Attribute names what style to use.

For example, the following HTML document links to the CSS file space.css, shown above. The first paragraph uses the spaceship style, and the second paragraph uses the moon style.

default.html
<!DOCTYPE html>
<html>
    <head>
        <title>Space</title>
        <link rel="stylesheet" href="space.css">
    </head>
    <body>
        <p class="spaceship">Atlantis</p>
        <p class="moon">Europa</p>        
    </body>
</html>

The following shows the effects of using the above styles. Move your mouse into the following box to see the effect of the styles.

Europa

Text Properties

The following CSS properties control how text is displayed on a page.

Font Family

The font-family property controls what font is used for text. The value of the font-family property is a set of font names, separated by commas (,). If a font name has a space in it, use double quotes around the name (").

If the first font is not found on the computer viewing the page, then the next font is tried, and so on until a font is found. Since different computers might have different fonts installed, it is usually good to choose fonts that tend to work on all computers. You can find a list by seaching for web safe fonts, or use one of the following links:

  • http://www.w3schools.com/cssref/css_websafe_fonts.asp
  • http://www.webdesigndev.com/web-development/16-gorgeous-web-safe-fonts-to-use-with-css

Example

Example

.example {
    font-family: "Lucida Sans Unicode", "Lucida Grande", sans-serif;
}
<p class="example">Example</p>

Font Size

The font-size property gives the size of the font.

Font sizes are typically specified in points, which are 1/72nd of an inch. To specify a size in points, write pt after the number.

Example

Example

    font-size: 24pt;
}
<p class="example">Example</p>

Color

The color property can be used to say what color to use for text. For controlling the color around the text, see Background Color.

You can select a named color here: http://w3schools.com/tags/ref_colornames.asp . For example, the following uses the color named red.

Example

Example

.example {
    color: red;
}
<p class="example">Example</p>

Or you can make your own color by:

  • Selecting a custom color: http://w3schools.com/tags/ref_colorpicker.asp
  • Mixing two colors: http://w3schools.com/tags/ref_colormixer.asp

These colors are numbers (shown in base 16, or hexadecimal) that are in the form #hhhhhh, where h is a hexadecimal digit (0-9 or A-F).

For example, the following uses the color #FF00FF:

Example

Example

.example {
    color: #ff00ff;
}
<p class="example">Example</p>

Color values are sometimes also written in the form rgb(r, g, b), where r, g, and b are integer numbers between 0 and 255 for how much red, green, and blue are in the color (0 being the darkest, and 255 being the brightest).

For example, the following uses the color rgb(255, 0, 0):

Example

Example

.example {
    color: rgb(255, 0, 0);
}
<p class="example">Example</p>

Background Properties

The following CSS properties control how the area around the content (text or image) of an HTML element is displayed. See the Box Properties for more information.

Most HTML Elements have a background. Most elements start out with a transparent (see through) background, meaning the background of the containing element is used.

The body element starts out with a background of white, which is why you start out with a white page.

Background Color

The background-color property gives a single color for the background. You can use the same kind of values for the Background Color property as you can with the Color Property.

For example, the following uses the named color yellow as the background.

Example

Example

.example {
    background-color: yellow;
}
<p class="example">Example</p>

Background Image

The background-image property uses an image to display the background. The value of the background-image property is the name of the image file.

For example, the following uses an image file named smiley.png. You can use the Background Repeat property to control whether the image is repeated.

smiley.png

Example

Example

.example {
    background-image: url("smiley.png");
}
<p class="example">Example</p>

Background Repeat

The background-repeat property controls how the image used with the Background Image property is repeated.

The following examples uses an image file named smiley.png as the background image.

smiley.png

This example does not repeat the image.

Example

Example

.example {
    background-image: url("smiley.png");
    background-repeat: no-repeat;
}
<p class="example">Example</p>

The following example repeats the image in the horizontal direction.

Example

Example

.example {
    background-image: url("smiley.png");
    background-repeat: repeat-x;
}
<p class="example">Example</p>

The following example repeats the image in the vertical direction.

Example

Example

.example {
    background-image: url("smiley.png");
    background-repeat: repeat-y;
}
<p class="example">Example</p>

Box Properties

Many HTML elements have box properties that control what is displayed around or behind the element content (text or image).

The box is the overall space on the page that an element takes up, and is made up of the following:

  • The margin is the amount of space that separates this element from other elements. This part is transparent. The size of the margin depends on the Margin Property.

    An element may have a margin even if you haven't set it.

  • The border is the amount of space that is taken up by the border. What the border looks like depends on on the Border Style Property. The size of the border depends on the Border Width Property.

    If you don't say what Border Style to use, there will be no border.

  • The padding is the amount of space to display around the text or image. The size of the padding depends on the Padding Property. What is displayed in this area is displayed depends on the Background Properties.

    An element may have padding even if you haven't set it.

Margin Property

The padding property is one way to put space around an element.

Read about Box Properties before using this property.

How to learn more

This class teaches a subset (part of) the CSS syntax. The subset was chosen both to make learning CSS easier, and to avoid practices that lead to CSS documents that are more difficult for others to understand and maintain. As you become a more seasoned CSS author you will learn more syntax, and some of the rules in this document can be relaxed. If you are curious, the full (and more precise) syntax for the CSS language is available https://developer.mozilla.org/en-US/docs/Web/CSS/CSS3.

You can usually find what you want by doing a web search on what you want as long as you include the word CSS. For example, if you wanted to know about how to set the font size, you could search for CSS font size. Although W3Schools is usually the first link, you may need to look at additional links to find what you need.

The mozilla developer site is a good resource for CSS: https://developer.mozilla.org/en-US/docs/Web/CSS.
