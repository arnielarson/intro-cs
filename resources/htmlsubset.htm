Introduction

HTML (HyperText Markup Language) is a language use to write down the contents of a web page.

Cascading Style Sheets (CSS) are typically used with HTML to provide the styling for a page.

Javascript is usually added to a page to control what happens when the page is used.

Where to put HTML files and how to name them

Name each file using a short name that describes what the file is for. You can use the section on Naming for guidance on picking names, and then add .html to the end you choose. Put all the files in the same folder. If there is a home page, name it default.html. For example:

default.html

typesOfSpaceships.html

moonBases.html

What an HTML looks like

An HTML file looks like the following:

<!DOCTYPE html>
<html>
    <head>
        Elements that go in the head
    </head>
    <body>
        Elements that go in the body
    </body>
</html>

Character References

Some characters are used by HTML for writing elements (for example, < and >), so we need to use a different way of writing these characters if we want to include them in the text of the document or in attribute values. The solution is to write a character reference as a substitute for the character. The following table gives some common character references:

Character Character Reference
< &lt;
> &gt;
& &amp;
" &quot;

It is only necessary to use an character reference for quote (") when it appears in an attribute value.

Elements that go in the <head>

The elements within the Head Element (abbreviated head) provide information about the Page. The following elements can appear within a Head Element.

  • Title Element (what the browser shows on the tab)
  • Link Element (to add a CSS file)
  • Script Element (to add a Javascript file)

Title Element

The Title Element (title) gives a short summary of the HTML document. The title of an HTML document is displayed outside of the body content for the page, typically on the tab for the page.

In the following example, the title is Lunar Lander.

<title>Lunar Lander</title>

Link Element

The Link Element in HTML can be used to add one or more Cascading Style Sheet (CSS) files to the page. The value of the href attribute should be the name of a CSS file in the same folder as the HTML document.

For more information on CSS, see the CSS Reference.

<link rel="stylesheet" href="file1.css">
<link rel="stylesheet" href="file2.css">

Script Element

The Script Element in HTML can be used to add one or more Javascript files to the page. The value of the src attribute should be the name of a Javascript file in the same folder as the HTML document.

For more information on Javascript, see the Javascript Reference.

The following example links to two Javascript files.

<script src="file1.js"></script>
<script src="file2.js"></script>

Elements that go in the <body>

The elements contained within the Body Element (body) determine what the web browser displays for the page. The following elements can appear within the Body Element:

  • Paragraph Element (for a paragraph)
  • Header Element (for the start of a section)
  • Division Element (to divide the page into parts)
  • Anchor Element (to make a hyperlink)
  • Image Element (to display an image)
  • Audio Element (to play a sound)
  • Table Element (for tables)

Paragraph Element

The Paragraph Element (abbreviated p) is used to separate paragraphs. The text and elements that appear within the paragraph make up the paragraph.

The following example shows two paragraphs containing text.

<p>This is some text in the first paragraph.</p>
<p>This is some text in the second paragraph.</p>

Header Elements

The header elements (abbreviated h1 through h6) represent the headings of a document. Headings can be used to organize a document into chapters, sections, etc. The first-level heading element (h1) is the most important sixth-level heading element (h6) is the least important.

Be careful not to confuse these header elements (used for headings) with the head element (which provides information about the page).

The heading element appear before the elements that make up the content of the heading. Heading elements typically contain text, but may also contain other elements.

The following example uses headings to represent two sections that each have two chapters.

<h1>Section 1/h1>
<h2>Chapter 1</h2>
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<h2>Chapter 2</h2>
<p>Paragraph 1</p>
<p>Paragraph 2</p>

<h1>Section 2/h1>
<h2>Chapter 1</h2>
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<h2>Chapter 2</h2>
<p>Paragraph 1</p>
<p>Paragraph 2</p>

Division Element

The division (abbreviated div) element is used to divide up (or group) portions of a document. You can think of it as a generic container for other elements.

Grouping elements together can be useful when styling a page (using CSS), since the style can be applied to the whole group.

In the following example, one division contains asteroids, and the other has spaceships:

<div>
    <p>Asteroid1</p>
    <p>Asteroid2</p>
    <p>Asteroid3</p>
</div>
<div>
    <p>Spaceship1</p>
    <p>Spaceship2</p>
    <p>Spaceship3</p>
</div>

Span Element

The Span (span) is used to create an element for a particular range (portion) of text.

A Span can useful when styling a page (using CSS), since a particular style can be applied to the text range. In the following example, a Span is used with a Class Attribute to specify that the words Moonbase and Asteroid Belt should be styled using whatever styling has been defined for the important style.

<p>
    The <span class="important">Moonbase</span> is next to the
    <span class="important">Asteroid Belt</span>.
</p>

A Span is also useful if you want to use or change a range of text from Javascript. In the following example, a Span is used with Identifier Attribute which is used by the start function to change the text of the span.

<p>
    The <span id="changeme">before</span> will be changed.
</p>
function start() {
    var spanElement = document.getElementById("changeme");
    spanElement.textContent = "after";
}

Anchor Element

An Anchor Element (abbreviated a) provides a hyperlink from the current page to another location. When the user clicks on the hyperlink, the web browser will navigate to that location. The text or elements contained within the Anchor Element are what the user should click on in order to navigate. The value of the href attribute indicates where to navigate, as follows:

  1. Link within page. If the value of the href attribute looks like #name, the browser will attempt to navigate to an element in the same HTML document that has the name. To name an element, use the Identifier Attribute. For example, the following Anchor Element will navigate to an element named moon.
    <p>Go to the <a href="#moon">Moon</a></p>
    <p id="moon">The Moon</p>
  2. Link to file. If the value of the href attribute is a file name, the browser will attempt to navigate to the file located in the same folder as the HTML document. For example, the following Anchor Element will navigate to a file called moonbase.html:
    <a href="moonbase.html">Moon Base</a>
  3. Link to URL. If the value of the href attribute is a URL, the browser will attempt to navigate to the resource located at that URL. For example, the following Anchor Element will navigate to the University Prep home page.
    <a href="http://www.universityprep.org/">University Prep</a>

Image Element

An Image Element (abbreviated img) displays an image. The source attribute (abbreviated src) names the image to display. The value of the source attribute should be the name of an image file that is in the same location as the HTML document.

An Image Element does not contain any elements or text.

The following example displays three images. There are many formats for representing images. The images in the following example are of type are:

  • PNG
  • JPEG
  • GIF

<img src="image1.png">
<img src="image1.jpg">
<img src="image1.gif">

Audio Element

An Audio Element is a way to add a sound to a page. Once you have the audio element on the page, then you can play (or replay) the sound.

The source attribute (abbreviated src) names the sound to display. The value of the source attribute should be the name of a audio file (like a MP3 file) that is in the same location as the HTML document.

<audio src="my.mp3" id="mysound"></audio>
function start() {
    var audioElement = document.getElementById("myaudio");
    
    // to start (or continue) playing
    audioElement.play();
    
    // to pause playing
    audioElement.pause();
    
    // to replay from the beginning
    audioElement.currentTime = 0;
    audioElement.play();
}

Table Element

An Table Element arranges text or other elements into a table. A table has a header part (abbreviated thead) which contains the names of the columns, and a body part (abbreviated tbody) which contains the data of the table.

Both the header part and body part can have multiple rows (abbreviated tr). Within a row, each cell is either a table header cell (abbreviated th) or a table data cell (abbreviated td).

The following example shows a table with two columns and three rows. The first column is named Letters and contains the letters A-C. The second column is named Numbers and contains the numbers 1-3.

<table>
    <thead>
        <tr>
            <th>Letters</th>
            <th>Numbers</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>A</td>
            <td>1</td>
        </tr>
        <tr>
            <td>B</td>
            <td>2</td>
        </tr>
        <tr>
            <td>C</td>
            <td>3</td>
        </tr>
    </tbody>
</table>

To add the rows using Javascript, use the innerHTML property of the table body element to add the HTML for the rows. For example:

<table>
    <thead>
        <tr>
            <th>Player</th>
            <th>Score</th>
        </tr>
    </thead>
    <tbody id="players">
    </tbody>
</table>        
var players = [
    {
        name: "Bob",
        score: 1
    },
    {
        name: "Cindy",
        score: 2
    },
    {
        name: "Fred",
        score: 0
    }
];

function start() {
    // get the table body element by giving its id
    var playersElement = document.getElementById("players");

    // for each player in the array
    var html = "";
    for (var i = 0; i < players.length; i = i + 1) {

        // get the player object
        var player = players[i];

        // add the HTML string for the row
        html = html + "<tr><td>" + player.name + "</td><td>" +
            player.score.toString() + "</td></tr>";
    }

    // change the HTML of the table body element
    playersElement.innerHTML = html;    
}

Attributes you can add to an element

An attribute is a way to add some extra information to an element. The following attributes can be added to an element:

  • Identifier Attribute (for naming an element)
  • Class Attribute (for choosing the style for an element)

Identifier Attribute

The Identifier Attribute (abbreviated id) gives a name to an element so it can be referred to elsewhere (for example, from an Anchor element, or from Javascript). Follow the guidance on Naming when choosing a value this attribute. This attribute does not affect the display of the element. The value of each id attribute must be unique within the HTML document. The case of identifier names matters (for example, aBC is different than abc).

In the following example, a paragraph element is named "nameOfMoon".

<p id="nameOfMoon">Saturn</p>

Class Attribute

The Class Attribute (class) is used to say what style is used for an element. The value of the Class Attribute should be the name of a style in a Cascading Style Sheet (CSS) document that has been linked to the HTML Document. See the Link Element for how to link a CSS document to an HTML Document. The case of style names matters (for example, aBC is different than abc).

In the following example, a paragraph containing the text "Legal Notice" is styled with a style named "warning".

<p class="warning">Read the following legal notice before continuing!</p>

To change the class of an element using Javascript, use the className property.

function start() {
    // get an element using its id
    var someElement = document.getElementById("someid");

    // if the element has the style named "hidden"
    if (someElement.className == "hidden") {
        // switch to the style named "visible"
        someElement.className = "visible";
    } else {
        // otherwise switch to the style named "hidden"
        someElement.className = "hidden";
    }
}

Additional information

Conventions

This class uses the following conventions when writing HTML documents:

  • Use lowercase for HTML tag names (<p> instead of <P>).
  • Use lowercase for HTML attribute names (src="moon.gif" intead of SRC="moon.gif").
  • Keep each line less than 120 characters to make the file it easy to read in an editor.
  • When an element is contained within another element, indent it with 4 spaces.
  • Follow the conventions described in Names.
  • We will use double (instead of single) quotes around attribute values.
  • Do not include blank lines unless you feel that it helps with readability.

Choosing names

The following conventions are used both for constructing names for Files and names for Identifiers within the HTML page.

Pick names that describe the purpose of what you are naming without being overly verbose. If a name requires multiple words, then put all the words together (no spaces), but only capitalize the first letter of the second and following words. For example:

moonbase

mainAsteroidBelt

When choosing a name, only use letters (a-z, A-Z), and avoid abbreviations.

How to learn more

This class teaches a subset (part of) the HTML syntax. The subset was chosen both to make learning HTML easier, and to avoid practices that lead to HTML documents that are more difficult for others to understand and maintain. As you become a more seasoned HTML author you will learn more syntax, and some of the rules in this document can be relaxed. If you are curious, the full (and more precise) syntax for the HTML language is available http://www.w3.org/html/wg/drafts/html/master/Overview.html.
